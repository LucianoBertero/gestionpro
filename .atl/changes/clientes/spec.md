# Delta for clientes

> New capabilities: `clientes-backend` + `clientes-frontend`. Full specs — no prior behavior.

## clientes-backend

| # | Requirement | MUST/SHALL | Scenarios (GIVEN→WHEN→THEN) |
|---|---|---|---|
| BE1 | **Cliente model**: CUIT único, denominación, encargadoId FK (required), supervisorId?, semáforo (EstadoSemaforo, default VERDE), actividades String[], domicilio?, teléfono?, email?, whatsapp?, condicionIva, termino (0\|3\|6), notas?, activo=true, estudioId=1. Soft-delete via activo:false. Relaciones: impuestos ClienteImpuesto[], encargado Usuario. | MUST | **Create with impuestos**: GIVEN valid DTO → WHEN POST /clientes → THEN Cliente + ClienteImpuesto records created in transaction. **Soft delete**: GIVEN activo=true → WHEN DELETE /:id → THEN activo=false, row preserved. **Duplicate CUIT**: GIVEN existing CUIT → WHEN POST → THEN 409 conflict. |
| BE2 | **ClienteImpuesto model**: clienteId FK, tipo (TipoImpuesto: AUTONOMOS\|IVA\|IIBB_LOCAL\|MUNICIPAL\|SUELDOS\|MONOTRIBUTO\|GANANCIAS), activo=true. Created alongside Cliente in create transaction. | MUST | **Impuestos in legajo**: GIVEN clienteId=1 → WHEN GET /clientes/1 → THEN impuestos array included. **Toggle impuesto**: GIVEN PATCH /clientes/:id with updated tipoImpuesto[] → WHEN save → THEN old inactivos, new created. |
| BE3 | **Lista paginada**: GET /clientes with ?pagina, ?porPagina, filters: encargadoId, semaforo, search (nombre\|CUIT). COLABORADOR auto-filtered WHERE encargadoId = authUser.userId. Response: { data, meta: { total, pagina, porPagina } }. | MUST | **SOCIO ve todo**: GIVEN SOCIO → WHEN GET → THEN all active clients. **COLABORADOR filtered**: GIVEN COLABORADOR userId=5 → WHEN GET → THEN only encargadoId=5 clients. **Pagination**: GIVEN 50 clients → WHEN ?pagina=2&porPagina=20 → THEN page 2, 20 items. **Search**: GIVEN search=García → WHEN GET → THEN matching denominación or CUIT. |
| BE4 | **Legajo**: GET /clientes/:id returns client + impuestos anidados + encargado {id, name}. SOCIO ve cualquier legajo. COLABORADOR solo si encargadoId=userId; otherwise 403. Deleted clients → 404. | MUST | **SOCIO access**: GIVEN SOCIO → WHEN GET /clientes/3 → THEN 200 + legajo. **COLABORADOR denied**: GIVEN COLABORADOR userId=5, encargadoId=8 → WHEN GET /clientes/8 → THEN 403. **Not found**: GIVEN deleted or nonexistent ID → WHEN GET → THEN 404. |
| BE5 | **AFIP autocomplete mock**: GET /clientes/afip/:cuit returns static sample: denominación, domicilio, condicionIva, actividades[]. Only valid CUIT format (XX-NNNNNNNN-X). Real AFIP integration deferred. | SHOULD | **Valid CUIT**: GIVEN 20-12345678-9 → WHEN GET → THEN mock data returned. **Invalid format**: GIVEN "abc" → WHEN GET → THEN 400. **Not found**: GIVEN unknown CUIT → WHEN GET → THEN 404 with descriptive message. |
| BE6 | **Semáforo stubbed**: Default VERDE on create. Field returned in all client responses. Real logic (overdue tasks + missing liquidations) deferred pending Tareas + Liquidaciones modules. | MUST | **Default value**: GIVEN new client → WHEN queried → THEN semaforo=VERDE. **Explicit set**: GIVEN PATCH semaforo=ROJO (SOCIO only) → WHEN save → THEN persisted. |
| BE7 | **RBAC controller split**: PublicController (JwtAuthGuard): GET lista, GET legajo, GET afip. AdminController (@AllowedRoles([SOCIO])): POST, PATCH /:id, DELETE /:id. PATCH/DELETE on deleted clients → 404. | MUST | **Public endpoints**: GIVEN valid JWT → WHEN GET endpoints → THEN 200. **Admin blocked**: GIVEN COLABORADOR → WHEN POST/PATCH/DELETE → THEN 403. **No JWT**: GIVEN no token → WHEN any endpoint → THEN 401. |

## clientes-frontend

| # | Requirement | MUST/SHALL | Scenarios (GIVEN→WHEN→THEN) |
|---|---|---|---|
| FE1 | **DataTable lista**: Columns: denominación, CUIT, encargado (name), semáforo (color chip), impuestos activos (badges), actions (edit/delete). Filters: encargado selector, semáforo selector, search input. Paginación via React Query + nuqs URL state. Row click navigates to legajo. | MUST | **Load**: GIVEN data → WHEN page loads → THEN table renders. **Filter**: GIVEN semáforo=ROJO selected → WHEN applied → THEN only ROJO rows, URL updated. **Navigate**: GIVEN row clicked → WHEN navigate → THEN /dashboard/clientes/[id]. |
| FE2 | **WhatsApp/Email quick actions**: WhatsApp button → wa.me/{numero}?text=Hola {denominacion} new tab. Email button → mailto:{email}?subject=GestiónPro. Hidden when field is null. | SHOULD | **WhatsApp**: GIVEN whatsapp non-null → WHEN click → THEN new tab wa.me. **Hidden**: GIVEN no email → THEN email button not rendered. |
| FE3 | **Create/Edit Sheet**: TanStack Form + Zod schema. Fields: cuit (triggers AFIP autocomplete on blur), denominación, encargadoId (UserPicker), condicionIva, termino, tipoImpuesto[] (multi-select), actividades (tag input), domicilio, teléfono, email, whatsapp, notas. On save: POST (create) or PATCH /:id (edit). Refetch list on success. SOCIO-only; COLABORADOR sees sheet button hidden. | MUST | **Create flow**: GIVEN form valid → WHEN save → THEN POST succeeds, sheet closes, list refetched. **Edit flow**: GIVEN edit clicked → WHEN form pre-filled + saved → THEN PATCH succeeds. **AFIP autocomplete**: GIVEN CUIT entered → WHEN blur → THEN denominación + domicilio + condicionIva auto-filled from GET /afip/:cuit. **Duplicate**: GIVEN CUIT duplicate → WHEN save → THEN error toast. |
| FE4 | **Legajo page**: /dashboard/clientes/[id] with tabs: Resumen (client info + impuestos, active), Tareas (stubbed "Próximamente"), Liquidaciones (stubbed), Archivos (stubbed), Comunicaciones (stubbed), Notas (stubbed). Delete button (SOCIO only). Back button to list. Loading/error states handled. | MUST | **Resumen active**: GIVEN legajo → THEN Resumen tab active with details. **Stubbed tabs**: GIVEN Tareas tab clicked → THEN "Próximamente" placeholder. **Delete**: GIVEN SOCIO → WHEN delete → THEN soft-delete, redirect to list. |
| FE5 | **Nav item RBAC**: "Clientes" item with Building icon in sidebar. Visible to SOCIO + COLABORADOR. Active state on /dashboard/clientes routes. Matches existing pattern (roles array in nav-config). | MUST | **Visible**: GIVEN SOCIO\|COLABORADOR → WHEN sidebar renders → THEN Clientes item shown. **Highlighted**: GIVEN /dashboard/clientes → THEN item marked active. |
